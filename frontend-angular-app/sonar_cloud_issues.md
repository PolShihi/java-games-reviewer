


## 🛡️ 1. `frontend-angular-app/src/app/core/http/base-url.interceptor.ts`
*   **Строка 11 (Уязвимость: MEDIUM):** `Make sure the regex used here... cannot lead to denial of service.`
*   **Категория:** ReDoS (Regular Expression Denial of Service)
    *   **В чем проблема:** Ты используешь регулярное выражение (Regex), скорее всего, чтобы проверить, начинается ли URL с `http://` или `https://` перед тем, как подставить базовый URL бэкенда. Твоя регулярка написана так (имеет вложенные повторения, например `(a+)+`), что при передаче специально сформированной длинной строки браузер "зависнет", пытаясь ее распарсить. Это называется катастрофический возврат (Catastrophic Backtracking).
    *   **Как исправить (в коде):** 
        Лучший способ избавиться от этой проблемы — вообще отказаться от сложных регулярных выражений для простых задач.
        
        *Было (примерно):*
        ```typescript
        const isAbsoluteUrl = /^https?:\/\/[a-z0-9\-._~:/?#[\]@!$&'()*+,;=.]+$/i.test(req.url);
        ```
        *Стало (простое и безопасное решение):*
        ```typescript
        const isAbsoluteUrl = req.url.startsWith('http://') || req.url.startsWith('https://');
        
        if (!isAbsoluteUrl) {
           // добавляем baseUrl
        }
        ```
    *   **Альтернативное решение (в SonarCloud):** Если ты уверен, что регулярка работает только с безопасными внутренними URL-адресами и юзер никак не может передать туда свой длинный текст, просто зайди в интерфейс SonarCloud, открой этот Hotspot и нажми кнопку **"Safe" (Безопасно)**.
