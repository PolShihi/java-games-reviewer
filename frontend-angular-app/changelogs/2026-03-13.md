# Changelog 2026-03-13

- Added Docker Compose stack for Postgres, backend, and both frontends in `docker-compose.yml`.
- Added Dockerfiles and nginx configs for `backend-app`, `frontend-react-app`, and `frontend-angular-app`.
- Added PostgreSQL init script in `docker/postgres/init.sql`.
- Added Docker ignore files for backend and frontends.
- Added `docker-commands.md` with common Docker workflows.
- Added root `.env` and wired Docker Compose to use environment variables.
- Added runtime Angular config via `assets/env.js` + `env.template.js` and nginx entrypoint for environment-based API URL.
- Reverted Docker Compose frontend API URL wiring to use `VITE_API_URL` as before.
- Set explicit dev server ports in frontend scripts (`vite --port 5173`, `ng serve --port 4200`).
- Added runtime env config for React (env.js + env.template.js + nginx envsubst) and wired container envs.
- Removed unused backend template files `application-prod.yml.template` and `application.yml.template`.
- Computed frontend API URLs in `docker-compose.yml` from `BACKEND_PORT`; removed `VITE_API_URL` from `.env`.
- Removed React build-time `VITE_API_URL` usage from Docker Compose and Dockerfile (runtime config only).
- Allowed React Docker build to include `docker-entrypoint.d` for env substitution.
- Reverted React Docker build context to `COPY . .` and kept `docker-entrypoint.d` in build context.
- Fixed React Docker build by including `nginx.conf` in the build context (removed from `.dockerignore`).
- Limited build-stage COPYs in React/Angular Dockerfiles to only required source files.
- Added `.prettierrc` to root `.gitignore`.
- Split env config into `.env` (public) + `.env.secrets` (secrets) with `.env.secrets.example`, wired via `env_file`.
- Unified database password usage by sourcing `DATABASE_PASSWORD` from `POSTGRES_PASSWORD`.
- Moved `POSTGRES_PASSWORD` out of `.env` into secrets.
- Removed `DATABASE_PASSWORD` interpolation from Docker Compose and documented it in `.env.secrets.example`.
- Ensured nginx envsubst entrypoint scripts are executable in React/Angular images.
- Included `src/assets` in Angular build assets so `env.template.js` is packaged.
- Adjusted Angular Dockerfile to serve the `browser` output folder so assets resolve correctly.
