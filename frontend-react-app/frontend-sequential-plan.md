# Frontend React: последовательный план работ

Дата создания: 2026-03-06
Проект: `frontend-react-app`

## 1. Зафиксировать API-контракт и базовую инфраструктуру
- Подтвердить backend base URL: `http://localhost:8080/api/v1` (в frontend сейчас стоит `8088`, это нужно синхронизировать на этапе реализации).
- Зафиксировать общий формат пагинации `PageResponse<T>` (`content`, `pageNumber`, `pageSize`, `totalElements`, `totalPages`, `first`, `last`).
- Утвердить единый обработчик ошибок для статусов `400/404/409/500` (включая валидационные ошибки с полем `errors`).
- Ввести централизованный API-слой: отдельные сервисы для `games`, `genres`, `production-companies`, `media-outlets`, `reviews`, `system-requirements`, `system-requirement-types`.

## 2. Маршрутизация и каркас страниц
- Настроить маршруты:
- `/` -> `GamesPage`
- `/games/new` -> `GameFormPage`
- `/games/:id` -> `GameDetailsPage`
- `/games/:id/edit` -> `GameFormPage` (режим редактирования)
- Добавить базовый layout (header + контейнер страницы + служебные уведомления).

## 3. Список игр (GamesPage)
- Реализовать серверную пагинацию, сортировку и загрузку данных через `GET /games`.
- Подключить переход в детали игры по клику на строку/кнопку.
- Добавить кнопку создания новой игры.
- Подготовить UI-состояния: `loading`, `empty`, `error`, `retry`.

## 4. Фильтрация игр
- Реализовать форму фильтров и запрос `GET /games/filter`:
- `title`, `yearFrom`, `yearTo`, `genreIds`, `developerId`, `publisherId`, `ratingFrom`, `ratingTo`
- Синхронизировать фильтры с пагинацией и сортировкой.
- (Опционально) синхронизировать состояние фильтров с query-параметрами URL.

## 5. Создание и редактирование игры (GameFormPage)
- Поднять форму на `react-hook-form`.
- Подтягивать справочники: `GET /genres`, `GET /production-companies`.
- Создание: `POST /games`.
- Редактирование: `GET /games/{id}` + `PUT /games/{id}`.
- Покрыть валидацию UI и обработку конфликтов (`409`) и ошибок валидации (`400`).

## 6. Страница деталей игры (GameDetailsPage)
- Загружать `GET /games/{id}` и отображать:
- базовую информацию (title, releaseYear, description, developer/publisher, genres)
- рейтинг
- системные требования
- обзоры
- Добавить действия: редактирование игры, удаление игры (`DELETE /games/{id}`) с подтверждением.

## 7. Модалка добавления системных требований
- Получать типы требований: `GET /system-requirement-types`.
- Форма модалки и создание через `POST /system-requirements`:
- `gameId`, `systemRequirementTypeId`, `storageGb`, `ramGb`, `cpuGhz`, `gpuTflops`, `vramGb`
- После успешного создания обновлять блок требований на странице деталей.

## 8. Модалка добавления обзора
- Получать список изданий: `GET /media-outlets`.
- Форма модалки и создание через `POST /reviews`:
- `gameId`, `mediaOutletId`, `score`, `summary`
- После успешного создания обновлять блок обзоров на странице деталей.

## 9. Финализация
- Проверить сценарий end-to-end:
- создание игры -> переход в детали -> добавление требования -> добавление обзора -> редактирование -> удаление.
- Прогнать `npm run lint` и исправить замечания.
- Подготовить README-разделы:
- запуск, env (`VITE_API_URL`), поддерживаемые страницы, ограничения.

## 10. Порядок реализации в этой сессии
- Сейчас выполнен только этап исследования и планирования.
- Дальше работа будет идти по запросу, шаг за шагом, без генерации всего проекта сразу.
