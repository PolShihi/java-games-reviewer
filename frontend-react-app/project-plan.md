Это важное дополнение. Действительно, игра без системных требований и обзоров в твоей базе данных не имеет смысла.

Поскольку у тебя связи **Один-ко-Многим** (одна игра -> много требований, одна игра -> много обзоров), лучшее место для управления ими — это **Страница деталей игры (`GameDetailsPage`)**.

Вот обновленные требования и план.

# Функциональные требования (Frontend v2.0)

## 1. Модуль "Игры" (Games)
### A. Список игр (Grid)
*   Таблица с пагинацией, сортировкой и фильтрами (как обсуждали ранее).
*   Отображение: ID, Название, Год, Разработчик, Рейтинг.

### B. Создание и Редактирование (Game Form)
*   Форма создания **самой сущности игры**: Название, Год, Описание, Выбор Разработчика/Издателя, Выбор Жанров.
*   *Важно:* Здесь мы создаем только "скелет" игры. Требования и обзоры добавляются позже на странице деталей (чтобы не перегружать форму создания).

### C. Детали игры (Game Details) — **Центр управления**
Это самая насыщенная страница. Она должна содержать:
1.  **Инфо-блок:** Обложка (плейсхолдер), описание, жанры тегами.
2.  **Блок "Системные требования" (System Requirements):**
    *   **Просмотр:** Таблица (Тип, ОС, CPU, GPU, RAM, HDD).
    *   **Действие:** Кнопка **"Добавить требование"**.
    *   **UI:** Открывается модальное окно (Dialog), где выбирается тип (Low/High) и вводятся цифры.
3.  **Блок "Обзоры критиков" (Reviews):**
    *   **Просмотр:** Список карточек (Имя издания, Оценка, Текст, Дата).
    *   **Действие:** Кнопка **"Написать обзор"**.
    *   **UI:** Модальное окно. Выпадающий список (Select) изданий (`media_outlets`), поле Оценка (Slider или Input 0-100), поле Текст.
4.  **Управление:** Кнопки "Редактировать игру" и "Удалить игру".

# Детальный план реализации (День за днем)

### День 1: База и Список
1.  Установка проекта и библиотек.
2.  Настройка `axiosConfig` и `Logger`.
3.  Создание `GameService.js` (метод `getAll`).
4.  Реализация `GamesPage.jsx` (Таблица с пагинацией).

### День 2: Фильтры и Создание игры
1.  Добавление фильтров (Search bar, Select Genre) на главную.
2.  Создание `GameFormPage.jsx`:
    *   Загрузка списка компаний и жанров в `useEffect`.
    *   Использование `react-hook-form` для валидации.
    *   Отправка `POST /games`.

### День 3: Страница Деталей (Просмотр)
1.  Создание `GameDetailsPage.jsx`.
2.  Запрос `GET /games/{id}`.
3.  Верстка:
    *   Сверху: Заголовок, описание, чипсы жанров.
    *   Снизу слева: Таблица системных требований (пока только чтение).
    *   Снизу справа: Список обзоров (пока только чтение).

### День 4: Добавление Системных требований
1.  **Backend:** Убедись, что есть эндпоинт `POST /system-requirements` (или вложенный в Game).
2.  **Frontend:**
    *   В `GameDetailsPage` добавь кнопку "Add Requirements".
    *   Создай компонент `AddRequirementDialog.jsx` (используй `MUI Dialog`).
    *   Внутри форма: Select (Type: Low/Med/High), Inputs (RAM, CPU, GPU).
    *   При сохранении: `SystemRequirementService.create(data)` -> обновить страницу деталей.

### День 5: Добавление Обзоров
1.  **Backend:** Убедись, что есть эндпоинт `POST /reviews`.
2.  **Frontend:**
    *   В `GameDetailsPage` добавь кнопку "Write Review".
    *   Создай компонент `AddReviewDialog.jsx`.
    *   **Важно:** Сначала нужно получить список изданий (`MediaOutlets`). Сделай `MediaOutletService.getAll()`.
    *   В модалке: Select (Outlet), Slider (Score 0-100), TextArea (Summary).
    *   При сохранении: `ReviewService.create(data)` -> обновить страницу.

### День 6: Тестирование и Чистка
1.  Проверь сценарий: Создать игру -> Зайти в детали -> Добавить требование -> Добавить обзор.
2.  Проверь удаление (игра удаляется вместе с обзорами?).
3.  Запусти `npm run lint` и исправь ошибки.

### Подсказка по реализации модальных окон (Dialog)

```javascript
const [openReviewDialog, setOpenReviewDialog] = useState(false);

// ... внутри return ...

<Button variant="contained" onClick={() => setOpenReviewDialog(true)}>
  Add Review
</Button>

<Dialog open={openReviewDialog} onClose={() => setOpenReviewDialog(false)}>
  <DialogTitle>Add New Review</DialogTitle>
  <DialogContent>
     {/* Твоя форма тут */}
  </DialogContent>
</Dialog>
```

Это самый профессиональный способ реализации вложенных форм.
